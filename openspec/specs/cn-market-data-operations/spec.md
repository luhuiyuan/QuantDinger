## Requirements
### Requirement: 管理员发起定向同步
系统必须（SHALL）提供仅管理员可调用的同步接口，接受受限标的集合、日期区间和允许的同步模式，返回持久化批次标识。接口不得默认启动全市场分钟数据导入，也不得由普通回测请求间接调用。

#### Scenario: 管理员同步指定标的
- **WHEN** 管理员提交两个规范 A 股标的及目标日线区间
- **THEN** 系统创建待执行批次并返回批次 ID，不阻塞 HTTP 请求等待全部完成

#### Scenario: 普通用户请求同步
- **WHEN** 非管理员调用同步或重试端点
- **THEN** 系统拒绝请求且不创建批次

### Requirement: 同步批次进度与重试
系统必须（SHALL）展示批次状态、标的总数、成功/运行/失败/跳过数量、当前标的、已覆盖区间、开始/结束时间和最近错误，并允许管理员只重试失败或不完整标的。重试必须复用安全检查点。

#### Scenario: 部分标的失败
- **WHEN** 批次中部分标的成功而另一个标的失败
- **THEN** 批次显示部分失败，成功标的数据保持可用，管理员可仅重试失败标的

### Requirement: 供应商健康状态
系统必须（SHALL）提供 TDX 候选节点的最近探测时间、延迟、连续失败、退避状态、最近成功时间及当前选中节点。健康检查不得在普通只读请求中产生无限或高并发探测。

#### Scenario: 所有节点不可用
- **WHEN** 所有候选节点均处于失败或退避状态
- **THEN** 管理接口显示供应商不可用，同步任务可重试失败且不得把目标覆盖标记为完成

### Requirement: 覆盖与质量运维视图
系统必须（SHALL）允许管理员按标的和日期查看最早/最晚数据、预期/实际交易日数、缺失区间、复权状态、未解决质量问题和最近成功同步，并支持从问题范围创建受限的修复同步请求。

#### Scenario: 查看有缺口的标的
- **WHEN** 管理员查询存在中间缺口的标的
- **THEN** API 和 Web 界面明确显示缺口，而不是只显示首尾覆盖日期

### Requirement: 磁盘空间保护
系统必须（SHALL）在创建扩大覆盖的同步任务及每个写入批次前检查配置的软硬空间阈值。达到软阈值时拒绝新的扩容任务；达到硬阈值时安全暂停当前任务并持久化检查点。系统不得自动删除项目数据、Docker 缓存或共享缓存。

#### Scenario: 启动前达到软阈值
- **WHEN** 可用磁盘空间低于软阈值但尚高于硬阈值
- **THEN** 新的回填任务被拒绝，状态接口返回阈值、实测空间和人工处理提示

#### Scenario: 执行中达到硬阈值
- **WHEN** 同步过程中可用空间降至硬阈值
- **THEN** 当前任务在安全提交边界暂停并保留可恢复检查点

### Requirement: 串行低资源执行
后台同步必须（SHALL）默认串行处理标的和分页，不启动并行编译、并行数据导入或不受限的任务扇出。单批读取和写入大小必须可配置并具有保守默认值。

#### Scenario: 大标的列表
- **WHEN** 管理员提交包含多个标的的同步批次
- **THEN** worker 按受控顺序逐个处理，不同时为多个标的保留完整历史数据于内存

### Requirement: 运维操作审计
系统必须（SHALL）记录同步创建、取消、重试和修复操作的操作者、时间、请求范围与结果；日志和数据库记录不得包含密码、API 密钥或其他秘密。

#### Scenario: 管理员重试失败批次
- **WHEN** 管理员发起重试
- **THEN** 系统保存操作者与重试范围，并将新尝试关联到原批次

### Requirement: 前端展示与兼容降级
Web 设置页必须（SHALL）展示供应商、覆盖、质量、批次和磁盘保护状态，并提供权限受控的定向同步操作；回测中心必须展示覆盖错误、数据来源和执行假设。前端连接旧后端时必须降级为不可用提示，不得伪造健康或完整状态。

#### Scenario: 新端点不可用
- **WHEN** 前端调用市场历史能力端点得到明确的未支持响应
- **THEN** 界面显示后端尚未启用该能力且禁用同步按钮
