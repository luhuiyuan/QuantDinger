## Requirements
### Requirement: A 股日线回测只读本地历史库
Strategy API V2 对 `CNStock` 日线回测必须（SHALL）从 PostgreSQL 历史库读取数据，不得在同一次回测中静默调用远程供应商补洞或混合多个供应商结果。其他市场及非支持周期保持现有数据加载边界。

#### Scenario: 本地数据完整
- **WHEN** 所有 A 股标的在用户区间和预热区间均具有合格本地数据
- **THEN** 回测仅使用该本地版本的数据执行

#### Scenario: 本地数据缺失
- **WHEN** 任一必要区间无法由本地历史库证明完整
- **THEN** 回测在执行策略代码前失败且不调用远程行情降级链

### Requirement: 完整校验用户区间和预热区间
系统必须（SHALL）分别计算策略预热所需日期和用户请求日期，并验证每个标的在两个区间的交易日覆盖、质量状态及复权因子覆盖。任何阻断缺口都必须返回结构化错误。

#### Scenario: 用户区间完整但预热不足
- **WHEN** 用户选择日期均有数据但策略声明的预热 bars 无法满足
- **THEN** 系统返回预热覆盖不足错误并说明最早所需日期

#### Scenario: 多标的中一个标的不完整
- **WHEN** 组合中至少一个标的存在阻断缺口
- **THEN** 整次回测失败并按标的列出缺失区间，不执行部分组合回测

### Requirement: 结构化覆盖错误
覆盖失败响应必须（SHALL）包含稳定错误码、市场、规范标的、请求区间、预热区间、缺失/异常区间、当前覆盖水位和允许的后续动作。不得只返回供应商异常字符串或空 DataFrame 错误。

#### Scenario: 管理员可执行补数
- **WHEN** 回测因可同步的数据缺口失败
- **THEN** 错误响应包含可用于管理员定向同步的规范标的和日期范围，但不自动启动同步

#### Scenario: 数据存在阻断质量问题
- **WHEN** 日期存在但质量状态不允许回测
- **THEN** 错误明确区分质量阻断与单纯缺失

### Requirement: 保留上海交易日语义
系统必须（SHALL）把 A 股日线索引作为 `Asia/Shanghai` 市场会话日期处理，并在数据库、pandas 加载、策略事件和结果序列之间保持同一日历日期。UTC 转换不得使日线落到前一日或后一日。

#### Scenario: 加载上海交易日
- **WHEN** 数据库交易日为 `2026-07-20`
- **THEN** 策略收到的日线事件仍标识为 `2026-07-20` 的上海市场会话

### Requirement: 数据来源与版本记录
每次成功的 A 股回测必须（SHALL）记录每个标的的数据供应商、数据内容版本、覆盖摘要、复权模式、因子版本、数据截止时间和市场规则版本，并在持久化结果及 API 响应中可读取。

#### Scenario: 对比两次回测来源
- **WHEN** 用户读取两个成功回测的详情
- **THEN** 用户能够判断两次回测是否使用相同历史数据和复权口径

### Requirement: 未收盘日线不得进入回测
系统必须（SHALL）只使用已确认收盘的交易日日线。请求结束日期包含当前未收盘交易日时，该日期不得被伪装为完整日线；系统应按覆盖策略明确截断或拒绝，并在响应中说明。

#### Scenario: 盘中请求当天结束的日线回测
- **WHEN** 上海市场尚未收盘且请求结束日期为当天
- **THEN** 系统不加载当天盘中 bar，并明确返回可用截止日或覆盖错误
