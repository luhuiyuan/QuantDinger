## 为什么

QuantDinger 已能识别 `CNStock` 标的，但历史 K 线仍依赖实时第三方降级链路，且只使用小型进程内缓存；上游返回被截断的时间范围时，系统还可能静默接受不完整数据。因此，A 股回测目前无法保证持久的数据覆盖、可复现的输入，也无法准确模拟中国市场的交易规则。

## 变更内容

- 在现有 `quantdinger` PostgreSQL 数据库中增加 A 股日线历史库；不新增 PostgreSQL、MongoDB 或 ClickHouse 服务。
- 引入锁定版本的 `easy_tdx`，作为原始日线及除权除息数据的主要采集器，并实现可恢复的 TDX 节点选择、分页、重试、节流及 `CNStock` 标识标准化。
- 持久化原始 OHLCV/成交额、公司行动、复权因子、同步批次、单标的进度、质量问题和覆盖摘要。
- Strategy API V2 的 A 股回测改为读取确定性的本地历史数据，完整校验用户选择区间和预热区间，保留上海交易日日期；数据不完整时返回结构化覆盖错误，不再静默使用部分远程结果。
- 增加基础 A 股仿真规则：普通股票仅做多、T+1 可卖数量、买入以 100 股为一手且允许清仓零股、停牌/无价格处理、按日期判断涨跌停，以及包含最低佣金、印花税和过户费的非对称费用。
- 增加管理员 API 和 Web 界面，用于查看供应商健康、覆盖率、质量状态、同步批次、磁盘保护状态并发起定向同步；在回测中心展示生效的数据与执行假设。
- 在内存和磁盘保护下串行扩展数据，先同步明确请求的标的和选定系统标的池，不导入全市场分钟线。

## 能力

### 新增能力

- `cn-market-history`：持久化 A 股原始日线、TDX 采集、标准化、覆盖校验、质量检查和复权查询视图。
- `cn-corporate-actions`：持久化除权除息事件，并确定性计算前复权/后复权因子。
- `cn-backtest-data`：Strategy API V2 本地历史优先加载、交易所本地日线日期、数据来源记录和失败关闭式覆盖校验。
- `cn-backtest-market-rules`：A 股持仓结算、交易单位、交易约束、涨跌停、停牌和费用计算。
- `cn-market-data-operations`：管理员同步、供应商健康、批次进度、覆盖报告、磁盘保护和前端运维控制。

### 修改能力

当前没有既有 OpenSpec 能力。本次以以上新增能力描述 QuantDinger 现有行为的扩展。

## 影响范围

- 后端依赖与打包：`backend_api_python/requirements.txt`、后端镜像构建，以及来自 `/home/quantadinger/easy_tdx` 的锁定版本可分发包。
- 数据库：在 `backend_api_python/migrations/init.sql` 中维护幂等基准 DDL，并增加可应用到现有 PostgreSQL 数据库的日期迁移。
- 后端服务：新增市场历史仓储、供应商、同步、质量模块和 Celery 维护任务；新增 A 股执行策略；修改 `CNStockDataSource` 和 Strategy API V2 加载器。
- API：增加需认证的管理员市场数据端点，以及结构化的回测覆盖/来源响应。
- 前端：修改 `/home/quantadinger/QuantDinger-Vue` 中的设置页市场数据运维界面和回测中心假设/覆盖展示。
- 运维：串行日常同步、数据库 advisory lock、可重试的单标的检查点、明确的磁盘空间安全阈值；不构建 Mobile/H5。
- 许可：复用 easy_tdx 或 QUANTAXIS 衍生代码时保留 MIT 署名；数据供应商使用权由部署运维方负责。
